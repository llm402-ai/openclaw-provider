# @llm402/openclaw-provider

OpenClaw provider plugin for [llm402.ai](https://llm402.ai) — 400+ AI models
with pay-per-request pricing via Bitcoin Lightning, USDC on Base, Cashu ecash,
or prepaid balance tokens.

**No accounts. No API keys. Just payment.**

> **Before you install**, read the [Security](#security) section. This
> plugin handles wallet keys and real money. Never install on shared systems,
> CI runners, or development sandboxes.

---

## Install

### macOS / Linux

```bash
# Inside your OpenClaw plugin directory:
npm install @llm402/openclaw-provider
```

### Windows

```powershell
# Run in PowerShell (not cmd.exe) with Node 22+:
npm install @llm402/openclaw-provider
```

> Windows note: file permissions differ from Unix. See
> [Security -> Key storage](#key-storage) for recommended
> BitLocker / ACL hardening.

### Requirements

- **Node.js**: v22 or later
- **OpenClaw**: v2026.4.12 or later (tested)
- **Disk**: ~260 KB for the plugin, ~44 MB for `viem` (x402 mode only)

## Quick start

### 1. CLI wallet setup (recommended)

The package ships a CLI for wallet management. No OpenClaw install required
to create a wallet and fund it.

```bash
# Create a wallet (generates nsec + EVM keypair, saved to ~/.llm402/wallet.json)
npx llm402-openclaw init

# Fund with 5000 sats via Lightning
npx llm402-openclaw fund 5000

# Check your balance
npx llm402-openclaw balance
```

`init` hides secrets by default. Set `LLM402_SHOW_SECRETS=1` to reveal
the nsec and config snippet:

```bash
LLM402_SHOW_SECRETS=1 npx llm402-openclaw init
```

### CLI commands

| Command         | Description |
|-----------------|-------------|
| `init`          | Create or load a wallet. Secrets hidden unless `LLM402_SHOW_SECRETS=1`. |
| `fund <amount>` | Print a BOLT11 Lightning invoice, poll for payment, auto-mint Cashu proofs. |
| `balance`       | Show Cashu balance per mint. Add `--check-usdc` for on-chain USDC. |
| `check-funding` | Resolve pending quotes from prior `fund` timeouts. |
| `sync`          | Opt-in: pull wallet state from Nostr relays (NIP-60). |

### CLI environment variables

| Variable              | Used by  | Purpose |
|-----------------------|----------|---------|
| `LLM402_SHOW_SECRETS` | `init`   | Set to `1` to reveal the generated `nsec` / EVM key and print a ready-to-paste config snippet. Hidden by default. |
| `LLM402_NSEC`         | `init`, `sync` | Import an existing Nostr `nsec1...` instead of generating one. If a wallet already exists with a different `nsec`, `init` aborts to prevent silent overwrite. |
| `LLM402_EVM_KEY`      | `init`   | Import an existing EVM private key (0x-prefixed hex) instead of generating one. |

Flags (`--nsec`, `--evm-key`) are rejected on purpose — secrets passed on the command line leak via `/proc/cmdline` and shell history. Use the env vars above. The full CLI reference is at [llm402.ai/docs#openclaw](https://llm402.ai/docs#openclaw).

### 2. Balance mode (simplest, no wallet)

Get a prepaid balance token from [llm402.ai/chat](https://llm402.ai/chat), then
configure the plugin in OpenClaw's settings:

```json
{
  "paymentMode": "balance",
  "balanceToken": "bal_your_token_here"
}
```

Zero payment latency. Works like a standard API key. No wallet needed.

### 3. Cashu mode (pay with ecash)

Pay with Cashu tokens. Requires an nsec (generated by `init` or brought from
any NIP-60-compatible wallet like [Minibits](https://minibits.cash) or
[Nutstash](https://nutstash.app)).

```json
{
  "paymentMode": "cashu",
  "cashuNsec": "nsec1..."
}
```

### 4. x402 mode (pay with USDC on Base)

Pay with stablecoin. Gasless EIP-3009 signing — no ETH needed for gas.

```json
{
  "paymentMode": "x402",
  "evmPrivateKey": "0x..."
}
```

Per-request hardcoded cap: **$5 USDC**. Session cap configurable via
`sessionBudgetUsdcCents` (default $50).

### 5. Lightning mode (pay with sats)

Pay L402 invoices by melting Cashu proofs. Requires a funded Cashu wallet.

```json
{
  "paymentMode": "lightning",
  "cashuNsec": "nsec1..."
}
```

Lightning is fallback-of-last-resort in the payment chain (Cashu -> x402 ->
Lightning) because melt adds 2-10s latency and a small mint fee.

## Payment mechanics

The `fund` command prints a BOLT11 Lightning invoice. Pay it with any Lightning
wallet (Alby, Phoenix, Zeus, Breez, etc.). Once the mint confirms payment, sats
are stored locally as Cashu ecash proofs at `~/.llm402/wallet.json`.

When the plugin makes inference requests, it pays llm402.ai by including Cashu
proofs (or USDC signatures, or a balance token) in the request header. Payment
is atomic per-request — you only pay for what you use.

Cashu is an implementation detail: from your perspective, you fund with
Lightning and spend on inference. The ecash layer handles denomination,
proof management, and change automatically.

If `fund` times out before the mint confirms, the pending quote is saved to
`~/.llm402/pending/`. Run `check-funding` to resolve it later.

---

## Budget controls

Each session enforces independent sats and USDC budgets. Payment is refused
before signing if either would be exceeded.

```json
{
  "maxRequestBudgetSats": 500,
  "sessionBudgetSats": 10000,
  "sessionBudgetUsdcCents": 5000
}
```

| Field                     | Default | Max       | Rail |
|---------------------------|---------|-----------|------|
| `maxRequestBudgetSats`    | 500     | 50,000    | sats |
| `sessionBudgetSats`       | 10,000  | 1,000,000 | sats |
| `sessionBudgetUsdcCents`  | 5,000   | 500,000   | USDC |

## How it works

### Balance mode

Adds `Authorization: Bearer bal_...` to every request. llm402.ai deducts
from your prepaid balance. No 402 round-trip needed — zero payment latency.

### Wallet modes (Cashu / x402 / Lightning)

Starts a local HTTP proxy on `127.0.0.1` that handles payment negotiation
transparently. OpenClaw points at the proxy URL; the proxy handles the
402-and-pay cycle against llm402.ai.

```
 OpenClaw  --1-->  proxy (127.0.0.1)  --2-->  llm402.ai
                            <--3-- 402 {price, x402, invoice}
                            --4--> retry + payment header
                            <--5-- 200 + SSE stream
```

This ClawRouter-pattern works around
[openclaw/openclaw#30484](https://github.com/openclaw/openclaw/issues/30484)
(OpenClaw treats 402 as a fatal billing error).

## Security

### Threat model

**You are trusted to run your own wallet.** This plugin:
- Stores wallet keys on disk at `~/.llm402/wallet.json` (0600 on Unix,
  default ACL on Windows)
- Runs an HTTP proxy on 127.0.0.1 gated by a per-session auth token
- Signs transactions locally; no remote key escrow

**You must NOT:**
- Install on a shared system (other OS users can read the wallet file)
- Install on a CI runner, Codespace, or dev container
- Use a main wallet — fund in small amounts (<=50,000 sats / <=$50)
- Install untrusted OpenClaw plugins alongside this one (co-resident
  plugins run in the same Node process and can read the wallet file)

See [SECURITY.md](./SECURITY.md) for the full threat model and rotation
procedures.

### Plugin protections

- **Localhost only** — proxy binds to `127.0.0.1`, never network-exposed
- **Pinned x402 targets** — USDC contract, `payTo` address, and network
  are hardcoded. A compromised llm402.ai server cannot redirect payments.
- **Per-request $5 USDC cap** — hardcoded, cannot be disabled
- **Cross-rail session caps** — sats and USDC tracked independently
- **Fetch snapshot** — `globalThis.fetch` captured at import time;
  monkey-patching by co-resident plugins has no effect
- **SSRF protection** — any overridden baseUrl validated against private-IP
  and localhost lists
- **Secret redaction** — all log output + error responses pass through
  `redactSecrets()` which strips Bearer tokens, macaroons, preimages,
  nsec, and EVM keys
- **User-Agent stamp** — every outbound request carries
  `llm402-openclaw-provider/<version>` so the server can segment traffic
  and deploy a kill-switch if a buggy version ships

### Key storage

Wallet at `~/.llm402/wallet.json`:
- **macOS / Linux**: 0600 (owner-only). Verify: `ls -l ~/.llm402/`
- **Windows**: default NTFS ACL (Owner + SYSTEM). Recommended:
  - Full-disk encryption via BitLocker
  - Or restrict ACL: `icacls %USERPROFILE%\.llm402 /inheritance:r /grant:r "%USERNAME%:(F)"`

To rotate: stop OpenClaw, delete `~/.llm402/wallet.json`, restart. A new
wallet is generated on next activation. Drain the old wallet first.

## Upgrading from 0.2.0

### Breaking changes

- **`@llm402/core` merged into this package.** The separate `@llm402/core`
  package is deprecated. Remove it from your dependencies:
  ```bash
  npm uninstall @llm402/core
  ```
  If you imported from `@llm402/core`, switch to
  `@llm402/openclaw-provider/lib`.

- **`checkFunding()` return type changed.** Returns `'paid' | 'expired' | 'unpaid'`
  instead of `boolean`. Update any `if (result === true)` checks to
  `if (result === 'paid')`.

- **`syncFromNostr()` signature changed.** Now takes an optional opts parameter:
  ```ts
  syncFromNostr(opts?: { relays?: string[]; allowRemoteMints?: boolean })
  ```
  Returns `{ events: number; proofsAdded: number; sats: number }`.

### Already done in 0.2.0

- `BESTPATH_*` environment variables are not used. This package has never
  used them, but if you are migrating from an older prototype, remove them.
